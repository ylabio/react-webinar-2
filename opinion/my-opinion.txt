Начиная изучать front-end, я сразу открыл для себя Redux и соответственно при решинии задач со store, мне ближе
использование именно этой технологии.
В рамках данного курса я познакомился с другим методом решения задач со store и мне этот способ показался очень непривычным.

В Redux можно выделить такие плюсы как наличие графического интерфейса для отслеживания состояния store, более компактная и
понятная запись(с выходом redux toolkit запись стала еще короче и удобнее)

Из минусов можно выделить наверно то, что это так или иначе библиотека которой нагружаем проект. Синтаксис Redux может меняться,
из-за чего может возникнуть сложность поддерживания legacy проектов, для работы с api требуется еще одна зависимость.

Из плюсов в использовании классового подхода можно выделить нативность и привязанность только к стандартам es
Нет привязанности к сторонним библиотекам, можно интегрировать и на legacy проекты с помощью babel

К минусам можно отнести непривычный подход в современном программировании, громоздкость записи

По поводу общей архитектуры могу сказать, что на react пишут все по разному, и стандарты оформления проекта везде разные
Лично для меня подход такого оформления скорее больше путает. Огромное количество папок, файлы стилей не централизованы,
3 разных названия(название папки, название компонента, название файла компонента)
